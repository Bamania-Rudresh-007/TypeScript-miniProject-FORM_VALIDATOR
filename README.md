# 📋 TypeSrcipt-learning-project — FORM_VALIDATOR

A lightweight, type-safe form validation utility built with **TypeScript**. It uses discriminated unions and type guards to validate user input and return structured success/error results — no third-party libraries required.

---

## 🚀 Features

- ✅ Full TypeScript type safety with discriminated union result types
- 🛡️ Runtime type guard (`isFormInput`) to validate unknown input shapes
- 📦 Zero dependencies
- 🔍 Multiple validation rules with descriptive error messages
- 🧪 Easy to extend with new fields or rules

## 🧠 How It Works

### Types

```ts
type FormInput = {
  name: string;
  email: string;
  age: number;
};

type ValidationSuccess = { success: true; data: FormInput };
type ValidationError   = { success: false; error: string[] };
type ValidationResult  = ValidationSuccess | ValidationError;
```

`ValidationResult` is a **discriminated union** — callers narrow the type via `result.success` and get full type inference on either branch.

### Validation Rules

| Field   | Rule                                              |
|---------|---------------------------------------------------|
| `name`  | Must not be empty or whitespace-only              |
| `email` | Must contain `@` and be fully lowercase           |
| `age`   | Must be 18 or older                               |
| (any)   | Input must be a non-null object with all 3 fields |

## 📌 Usage

```ts
validate({ name: "Rudresh", email: "rudresh@gmail.com", age: 18 });
// ✅ Success! Registered user: Rudresh

validate({ name: "", email: "Example@gmail.com", age: 16 });
// ❌ Validation Failed: Name cannot be empty., Invalid email format., Must be at least 18 years old.

validate("Not an object");
// ❌ Validation Failed: Invalid data structure or missing fields.

validate({ name: "Dharmik", email: "dharmik@test.com" });
// ❌ Validation Failed: Invalid data structure or missing fields.
```

## ⚙️ Running the Project

### Prerequisites

- [Node.js](https://nodejs.org/) (v16+)
- [TypeScript](https://www.typescriptlang.org/) (`npm install -g typescript`)

### Steps

```bash
# 1. Clone the repository
git clone https://github.com/your-username/TypeSrcipt-learning-project.git
cd TypeSrcipt-learning-project

# 2. Install dependencies
npm install

# 3. Compile TypeScript
npx tsc

# 4. Run the compiled output
node dist/index.js
```

Or use `ts-node` for a one-step run:

```bash
npm install -g ts-node
ts-node src/index.ts
```

## 🔧 Extending the Validator

To add a new field or rule, update `FormInput`, `isFormInput`, and `validateInput`:

```ts
// 1. Add field to the type
type FormInput = { name: string; email: string; age: number; phone: string };

// 2. Add runtime check
typeof input.phone === "string"

// 3. Add validation rule
if (!/^\d{10}$/.test(input.phone)) error.push("Phone must be 10 digits.");
```

## 📚 Concepts Demonstrated

- **Discriminated unions** — type-safe result handling without exceptions
- **Type guards** — narrowing `unknown` input at runtime
- **TypeScript utility types** — composing reusable, readable type definitions
- **Separation of concerns** — type checking, validation logic, and output each in their own function
